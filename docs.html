RAG Document Chatbot

Upload any PDF and chat with it using AI. Built with LangChain, LangGraph Agents, FastAPI, and React.

🗺️ Overview

This project is a full-stack AI application that lets you upload a PDF document and ask questions about it using natural language. The AI reads only your document — it doesn't make things up.

📄 Upload a PDF
The document is split into chunks, embedded into vectors, and stored locally in ChromaDB.
💬 Ask Questions
Your question is matched against the document chunks. Relevant pieces are sent to the AI.
🤖 AI Answers
The LLM reads the retrieved chunks and generates an accurate, grounded answer.
🔀 Pick Your Model
Choose from Groq or OpenRouter free models via a dropdown in the UI.

💡 Core Concepts

Before diving into the code, here are the key ideas you need to understand:

What is RAG?

RAG stands for Retrieval-Augmented Generation. It solves a simple problem: LLMs don't know about your documents. RAG gives them access by:

  1. Breaking your document into small chunks
  2. Converting each chunk into a vector (a list of numbers that captures meaning)
  3. When you ask a question, finding the chunks most similar to your question
  4. Sending those chunks to the LLM as context
💡 Think of it like giving the AI a "cheat sheet" from your document instead of asking it to memorize everything.

What is a Vector / Embedding?

An embedding converts text into a list of numbers. Similar meanings produce similar numbers. This lets us do semantic search — find text by meaning, not just keywords.

# "What is the revenue?" → [0.23, -0.51, 0.88, ...]
# "How much money did we make?" → [0.21, -0.49, 0.85, ...]
# These are close in vector space → both match "revenue" chunks

What is ChromaDB?

ChromaDB is a vector database — it stores embeddings and lets you search them by similarity. It runs entirely locally on your machine (no cloud, no account).

What is LangChain?

LangChain is a framework that connects LLMs to everything: document loaders, vector stores, tools, APIs. Instead of writing 100 lines of glue code, LangChain gives you pre-built components.

What is LangGraph?

LangGraph is built on top of LangChain. It lets you build agents — AI systems that can decide what steps to take, call tools, and loop until they have a final answer.

The code — rag.py

# Split the PDF into chunks
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(docs)

# Embed and store in Chroma
vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2"),
    persist_directory="chroma_db",
)

# At query time — find the 4 most relevant chunks
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})
docs = retriever.invoke("What is the revenue?")

🤖 How Agents Work

An agent is an AI system that can decide what to do instead of following a fixed script. You give it tools, and it figures out which ones to call.

The ReAct Pattern

This project uses the ReAct pattern (Reason + Act). The LLM thinks step by step:

User: "Who wrote this document?"

Thought: I need to search the document for author information
Action: search_document
Action Input: "author"
Observation: "Written by John Smith, Senior Analyst..."

Thought: I found the author
Final Answer: The document was written by John Smith.

Agent Tools (Skills)

Our agent has 2 tools defined in skills/document_tools.py:

ToolWhat it doesWhen it's used
search_document Searches ChromaDB for chunks matching the query Almost every question about the document
list_docs Lists all uploaded document names When user asks "what files do I have?"

The code — agents/rag_agent.py

# Define a tool using the @tool decorator
@tool
def search_document(query: str) -> str:
    """Search the uploaded PDF for relevant content."""
    retriever = get_retriever(collection_name)
    docs = retriever.invoke(query)
    return "\n\n".join([d.page_content for d in docs])

# Create the ReAct agent using LangGraph
agent = create_react_agent(model=llm, tools=[search_document, list_docs])

# Run it
result = agent.invoke({"messages": [{"role": "user", "content": user_message}]})
answer = result["messages"][-1].content

⚡ Fast vs Agent Mode

The chat UI has a toggle button in the bottom-right. Here's the difference:

⚡ Fast Mode Default
Flow: question → retrieve chunks → LLM → answer

LLM calls: 1
Speed: 3–8 seconds
Best for: Simple questions about the document
🤖 Agent Mode Advanced
Flow: question → agent thinks → calls tools → answers

LLM calls: 2–4
Speed: 15–40 seconds
Best for: Complex, multi-step questions

🚀 Local Setup

Get API Keys (free)
• Groq: console.groq.com → Sign up → API Keys
• OpenRouter: openrouter.ai/keys → Sign up → Create key
Create the .env file
Inside backend/, create a file named .env:
GROQ_API_KEY=your_groq_key_here
OPENROUTER_API_KEY=your_openrouter_key_here
Start the Backend
Double-click start_backend.bat or run:
cd backend
../venv/Scripts/activate
uvicorn main:app --reload --port 8000
You should see: INFO: Uvicorn running on http://127.0.0.1:8000
Start the Frontend
Double-click start_frontend.bat or run:
cd frontend
npm run dev
Open the app
Go to http://localhost:5173 in your browser. Upload a PDF, select a model, and start chatting.

📡 API Reference

The backend runs on http://localhost:8000. All endpoints:

GET /health Check if backend is running

GET /models List available models 
{
  "models": [
    { "id": "llama-3.1-8b-instant", "provider": "groq" },
    { "id": "z-ai/glm-4.5-air:free", "provider": "openrouter" }
  ]
}
POST /upload Upload and index a PDF

Request: multipart/form-data with file field

{
  "message": "Uploaded and indexed 'report.pdf'",
  "chunks": 42,
  "collection": "report"
}
POST /chat Send a message, get an answer

Request body:

{
  "message": "What is this document about?",
  "model_id": "llama-3.1-8b-instant",
  "document_name": "report.pdf",
  "mode": "fast"  // or "agent"
}

Response:

{ "answer": "This document is about..." }
